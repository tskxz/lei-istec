Para resolver o **Exercício 1** (implementar RADIUS e depois TACACS+), os comandos exatos que deve introduzir na Linha de Comandos (**CLI**) do Router estão divididos em duas partes.

Antes de começar, certifique-se de que entra no modo de configuração global do Router executando os comandos:

```text
Router> enable
Router# configure terminal

```

Aqui estão os comandos para cada um dos cenários:

---

### 1. Comandos para o Servidor RADIUS

Introduza a seguinte sequência de comandos para ativar o AAA e configurar o protocolo RADIUS:

```text
! 1. Ativa o modelo AAA (Authentication, Authorization, and Accounting) no router
aaa new-model

! 2. Define o IP do servidor RADIUS (192.168.1.1), a porta (1645) e a palavra-passe/chave secreta (ISTEC)
radius-server host 192.168.1.1 auth-port 1645 key ISTEC

! 3. Cria uma regra de autenticação por omissão (default) que consulta primeiro o RADIUS e, se falhar, a base local
aaa authentication login default group radius local

! 4. Entra na configuração das linhas de acesso remoto (Telnet/SSH)
line vty 0 15

! 5. Aplica a regra de autenticação por omissão criada anteriormente a estas linhas
login authentication default

! 6. Sai da configuração de linha
exit

```

---

### 2. Comandos para o Servidor TACACS+

Para repetir o exercício mudando para o protocolo TACACS+, deve introduzir os seguintes comandos (substituindo a lógica anterior):

```text
! 1. Garante que o modelo AAA continua ativo (caso tenha reiniciado o router)
aaa new-model

! 2. Define o IP do servidor TACACS+ (192.168.1.1) e a respetiva chave secreta (ex: ISTEC_TACACS)
tacacs-server host 192.168.1.1 key ISTEC_TACACS

! 3. Altera a regra de autenticação por omissão para passar a utilizar o grupo TACACS+ em vez do RADIUS
aaa authentication login default group tacacs+ local

! 4. Aplica a nova configuração às linhas de acesso remoto (VTY)
line vty 0 15
login authentication default
exit

```

---

### 💡 Comando Adicional Recomendado (Prevenção de Bloqueio)

Como configurou o parâmetro `local` no final dos comandos de autenticação (`group radius local` / `group tacacs+ local`), o router irá procurar um utilizador local caso o servidor falhe. Por segurança, configure também um utilizador local diretamente no router para nunca perder o acesso:

```text
Router(config)# username admin password cisco

```

Calma, não te preocupes! Esse erro é clássico e acontece por um detalhe muito simples: o router está a tentar comunicar com o servidor, mas os dois ainda não se conseguem "ver" na rede.

Olhando para a tua topologia nas imagens, faltam afinar **dois detalhes cruciais** para que a autenticação funcione:

---

### 1. Dar IPs às portas do Router e do PC (O grande motivo de dar timeout)

Neste momento tens os cabos ligados, mas as portas do Router estão vermelhas (desligadas) e sem endereço IP. Vamos resolver isso.

**No Router (Clica no Router -> CLI) introduz estes comandos:**

```text
Router> enable
Router# configure terminal

! Entrar na interface ligada ao Switch (pela imagem parece ser a GigabitEthernet0/0)
Router(config)# interface g0/0
Router(config-if)# ip address 192.168.1.254 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

```

*(Nota: Se o cabo estiver ligado na porta `g0/1` em vez da `g0/0`, muda o comando para `interface g0/1`).* Assim que fizeres o `no shutdown`, as luzes do cabo vão passar a verde.

**No PC de teste:**

1. Clica no PC -> **Desktop** -> **IP Configuration**.
2. Configura assim:
* **IP Address:** `192.168.1.2`
* **Subnet Mask:** `255.255.255.0`
* **Default Gateway:** `192.168.1.254` (O IP do router)



---

### 2. Corrigir o IP do Servidor AAA

Na tua segunda imagem (onde configuraste o RADIUS no Server0), preencheste o campo **Client IP** com `192.168.1.1`.

**O erro está aí:** o "Client IP" dentro do servidor AAA tem de ser o **IP do Router** (que é quem está a pedir a autenticação), e não o IP do próprio servidor.

Vai ao **Server0** -> **Services** -> **AAA** e corrige:

1. Clica no cliente que criaste na tabela em baixo para editá-lo (ou apaga e cria de novo).
2. **Client Name:** `Router`
3. **Client IP:** `192.168.1.254` *(Aqui está o segredo! Tem de ser o IP do Router)*
4. **Secret:** `ISTEC`
5. **Server Type:** `RADIUS`
6. Clica em **Save** ou **Add**.

### O Teste Final (A Prova Real)

Antes de tentares o Telnet outra vez, vai ao **PC** -> **Command Prompt** e digita:

```bash
ping 192.168.1.254

```

Se o ping responder com sucesso, significa que a rede está operacional.

Agora sim, podes correr o comando `telnet 192.168.1.254`. Ele vai pedir o username, e quando digitares `user_radius` e `pass123`, o servidor vai deixar-te entrar direto!

Boa, isso significa que a rede já está a funcionar a 100%! O PC já consegue falar com o Router e o Router já está a pedir a autenticação.

Se te está a dar **"Login invalid"**, é porque as credenciais que estás a digitar não coincidem exatamente com o que está configurado no **Server0 (Servidor AAA)** ou porque o servidor rejeitou o pedido por causa da "Secret Key".

Tenta meter o seguinte utilizador (que é o que configurámos para o RADIUS):

* **Username:** `user_radius`
* **Password:** `pass123`
*(Nota: Quando digitas a password no terminal, não vai aparecer nenhuma letra ou asterisco por motivos de segurança. É normal! Digita `pass123` e carrega no Enter).*

### Se continuar a dar "Login invalid", como resolver?

Se mesmo assim falhar, há um pequeno erro de configuração entre o Router e o Servidor. Verifica estes dois pontos rápidos:

#### 1. No Servidor (Server0) -> Services -> AAA

Clica no cliente que tens listado na tabela em baixo para ver se os dados estão exatamente assim:

* **Client IP:** Tem de ser `192.168.1.254` (O IP do Router).
* **Secret:** Tem de ser `ISTEC` (Em maiúsculas).
* **User Setup (em baixo):** Confirma se o utilizador `user_radius` com a password `pass123` foi mesmo adicionado à lista (clica no botão "+" ou "Add").

#### 2. No Router (CLI)

A chave secreta (Secret) no Router tem de ser igualzinha à do servidor. Corre estes comandos no Router para garantir que está certo:

```text
Router> enable
Router# configure terminal
Router(config)# radius-server host 192.168.1.1 auth-port 1645 key ISTEC
Router(config)# exit

```

Tenta novamente o login no PC. Qual foi o resultado desta vez?

**Não, não era suposto fechar a ligação.** Esse comportamento significa que o Router tentou contactar o servidor RADIUS mas não obteve resposta a tempo (deu *timeout*) ou houve uma falha de autenticação severa, fazendo com que ele desligasse a sessão Telnet automaticamente.

Vamos resolver isso rapidamente. O problema está quase de certeza numa destas **três coisas** no Packet Tracer:

### 1. Falta de Password de "Enable" no Router (Muito Comum)

Por segurança, os routers Cisco não permitem ligações remotas (Telnet/SSH) se o router não tiver uma palavra-passe configurada para o modo privilegiado (`enable`). Se não existir, ele deita a ligação abaixo assim que fazes login.

**Como resolver:**
Vai à CLI do **Router** e define uma password de enable:

```text
Router> enable
Router# configure terminal
Router(config)# enable secret cisco
Router(config)# exit

```

*(A partir de agora, a password para entrar no modo `enable` será `cisco`).*

---

### 2. O IP do Servidor AAA está incorreto na configuração do Router

Se o Router estiver a enviar o pedido para o IP errado, ele nunca vai receber resposta.

**Como verificar e corrigir:**
Confirma qual é o IP que deste ao **Server0** (no Desktop -> IP Configuration).

* Se o IP do servidor for `192.168.1.1`, o comando no Router tem de estar exatamente assim (com a porta 1645):

```text
Router> enable
Router# configure terminal
Router(config)# radius-server host 192.168.1.1 auth-port 1645 key ISTEC
Router(config)# exit

```

---

### 3. O Servidor não sabe para onde responder (Falta o Default Gateway no Server0)

Para o Servidor AAA conseguir responder ao Router, ele precisa de saber quem é o "caminho de saída".

1. Clica no **Server0**.
2. Vai a **Desktop** -> **IP Configuration**.
3. Garante que o campo **Default Gateway** tem o IP do Router: `192.168.1.254`.

---

3. Introduz `user_radius` e a password `pass123`.

Desta vez, em vez de fechar a ligação, ele deverá mostrar:
